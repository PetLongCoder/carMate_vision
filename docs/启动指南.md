# CarMate 项目启动指南

## 快速启动（推荐）

双击根目录下的 `start_all.bat`，或在终端中运行：

```powershell
cd 项目根目录
.\start_all.bat
```

脚本会自动完成以下操作：

1. **环境检查** — 检测 MediaMTX、FFmpeg、Python、Node.js 是否就绪
2. **清理旧进程** — 关闭残留的 MediaMTX、后端（端口 8000）、前端（端口 5173）
3. **启动服务**：MediaMTX → FastAPI 后端 → Vite 前端（依次启动）
4. **显示访问地址**

> 首次运行前请确保已安装依赖，详见下方 **首次运行准备**。

## 首次运行准备

### 后端依赖

```powershell
cd backend
pip install -r requirements.txt
```

首次启动会自动从 `.env.example` 创建 `.env`，无需手动配置。

### 前端依赖

```powershell
cd frontend
npm install
```

### FFmpeg（可选，用于推流测试）

确认 FFmpeg 已加入系统 PATH。验证方式：

```powershell
ffmpeg -version
```

## 访问地址

启动后各服务地址如下：

| 地址                             | 内容            |
| -------------------------------- | --------------- |
| http://localhost:5173            | 前端页面        |
| http://localhost:8000/docs       | API 接口文档    |
| http://localhost:8000/api/health | 健康检查        |
| http://localhost:8888            | HLS 视频播放    |
| http://localhost:8889            | WebRTC 视频播放 |

## 沙盘摄像头

支持以下 RTSP 摄像头地址（沙盘环境）：

```
rtsp://10.126.59.120:8554/live/live1
rtsp://10.126.59.120:8554/live/live2
...
rtsp://10.126.59.120:8554/live/live12
```

## 手动启动（逐个服务）

如需单独启动各服务进行调试：

### 1. MediaMTX

```powershell
cd mediamtx
.\mediamtx.exe mediamtx.yml
```

### 2. 后端

```powershell
cd backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 3. 前端

```powershell
cd frontend
npx vite --host 0.0.0.0 --port 5173
```

## 推流测试

```powershell
ffmpeg -re -stream_loop -1 -i test_video.mp4 -c copy -f rtsp rtsp://localhost:8554/camera
```

## 注意事项

- 确保端口 **8000、5173、8554、8888、8889** 未被占用
- Windows 防火墙可能会拦截 MediaMTX 端口，首次运行请允许访问
- 如需停止所有服务，直接关闭启动脚本的命令行窗口即可
