启动步骤
1
克隆仓库：git clone https://github.com/PetLongCoder/carMate_vision.git
2
进入前端目录：cd carMate_vision/frontend
3

4

5
启动开发服务器：npm.cmd run dev
6
浏览器访问：http://localhost:5173
2
环境配置

在 frontend/.env 文件中配置以下参数：

VITE_API_BASE_URL=http://localhost:8000/api VITE_WS_URL=ws://localhost:8000/ws VITE_USE_MOCK_AUTH=true VITE_ENABLE_WS=false
MOCK=true 使用本地 Mock 登录，无需后端，适合开发与功能演示
MOCK=false 连接真实后端 /auth/* 认证接口
WS=false 关闭 WebSocket，避免后端未启动时控制台报错
WS=true 开启实时推送，需后端 WebSocket 服务支持
⚠️ 修改 .env 文件后，需要重新执行 npm.cmd run dev 才能生效。
3
登录说明

未登录访问任意页面将自动跳转至 /login。登录成功后，普通用户进入车牌识别页，管理员进入控制面板。

普通用户 · 三种登录方式

选择「普通用户」标签，通过顶部分段按钮切换：

🔑

账号密码

输入用户名和密码，点击登录

📱

手机验证码

输入已注册手机号，获取验证码后登录

📧

邮箱验证码

输入已注册邮箱，获取验证码后登录

管理员登录

切换到「管理员」标签，仅支持账号密码登录。登录后可访问控制面板与告警中心。

内置测试账号（Mock 模式）

角色用户名密码手机号邮箱

💡 Mock 模式下验证码不会真实发送。点击「获取验证码」后，按 F12 打开浏览器控制台（Console），查找 [Mock SMS] 或 [Mock Email] 开头的 6 位验证码。
4
注册说明

注册入口：登录页底部「立即注册」，或访问 /register。仅面向普通用户，分两步完成：

第一步 · 验证手机号

输入手机号 → 获取验证码 → 填写验证码 → 点击「下一步」

→

第二步 · 设置账号

设置用户名、密码（邮箱可选）→ 点击「完成注册」→ 自动登录

业务规则

  • 未注册的手机号、邮箱、用户名不能用于登录
  • 已注册的手机号、邮箱、用户名不能重复注册
  • 出现冲突时弹窗询问是否前往登录或注册页，不自动跳转
  • 仅注册时填写了邮箱的账号，才能使用邮箱验证码登录
  • 管理员账号由系统预置，不能通过注册页面创建

后端联调接口

后端就绪后，将 VITE_USE_MOCK_AUTH 改为 false 并重启前端。前端已预留以下接口：

功能方法路径
账号密码登录POST/auth/login
用户注册POST/auth/register
发送短信验证码POST/auth/sms/send
发送邮箱验证码POST/auth/email/send
手机验证码登录POST/auth/sms/login
邮箱验证码登录POST/auth/email/login
获取当前用户GET/auth/me
7
常见问题
npm 命令无法执行？
Windows PowerShell 下请使用 npm.cmd 代替 npm，或换用 CMD 终端。
控制台一直报 WebSocket 连接错误？
将 .env 中 VITE_ENABLE_WS 设为 false，保存后重新执行 npm.cmd run dev。
验证码在哪里查看？
Mock 模式下按 F12 打开浏览器 Console 查看，验证码不会真实发送到手机或邮箱。
普通用户为什么进不了控制面板？
控制面板为管理员专属页面。普通用户登录后默认进入车牌识别页，属于正常的权限设计。
